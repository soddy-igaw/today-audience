# 오늘의 오디언스

> 라이브 오디언스 기획에 며칠 걸리던 일을, 90초 만에 끝내는 자동화 시스템

🔗 **라이브 데모**: https://soddy-igaw.github.io/today-audience/

## ① 프로젝트 소개

**오늘의 오디언스**는 매일 아침 자동으로 실행되는 오디언스 기획 파이프라인입니다.

실시간 뉴스 트렌드를 수집하고, IGAWorks DMP의 행동 데이터를 AWS Athena로 교차 분석해, 광고주가 바로 쓸 수 있는 오디언스를 에세이 형태로 자동 생성·배포합니다.

### 90초 안에 일어나는 일

```
뉴스 트렌드 자동 수집 (30초)
    → 13개 카테고리 중 가장 뜨거운 주제 자동 선정
    → AWS Athena로 DMP 교차 시그널 ADID 실측 (1분)
    → Claude API로 행동 기반 오디언스 에세이 생성
    → 정적 사이트 빌드 → GitHub Pages 자동 배포
```

사람 개입 없이, 매일 아침 새로운 오디언스가 라이브 페이지에 올라갑니다.

### 구성 요소

| 파일 | 역할 |
|---|---|
| `daily_publish.py` | 자동 발행 파이프라인 — 트렌드 수집, Athena 실측, LLM 에세이 생성, 코드 반영 |
| `build_static.py` | GitHub Pages용 정적 사이트 빌드 |
| `app.py` | Streamlit 대시보드 (내부 프리뷰) |
| `run_daily.sh` | 크론잡 실행 스크립트 (매일 10시) |
| `templates/` | 에세이 3타입 HTML 템플릿 (감정형·인사이트형·데이터형) |

### 커버 카테고리 (13개)

금융, 부동산, 증권, 쇼핑, 여행, 헬스, 자동차, 게임, 골프, 반려동물, 교육, 패션, 배달

## ② 문제 정의

### 라이브 오디언스 기획, 왜 며칠씩 걸리나

광고 캠페인의 핵심은 "누구에게 보여줄 것인가"입니다. 이 오디언스를 기획하는 과정에서 반복되는 병목:

1. **시간**: 트렌드 리서치 → DMP 분석 → 제안서 작성까지 **최소 2~3일, 복잡하면 일주일**. 기획자, 데이터 분석가, 디자이너가 각각 투입됩니다.

2. **타이밍**: 뉴스가 터져도 오디언스 제안까지 3~5일 걸려서, 트렌드가 식은 뒤에야 리포트가 나옵니다.

3. **추상적 타겟**: "2030 여성", "관심사: 여행" 같은 인구통계 기반 타겟은 실제 전환율이 낮습니다.

4. **데이터 단절**: 트렌드를 읽는 사람과 DMP 데이터를 다루는 사람이 분리되어 있어, 교차 인사이트가 나오기 어렵습니다.

### 이 프로젝트의 해결

뉴스 트렌드 → DMP 행동 시그널 → 오디언스 제안을 **하나의 자동화 파이프라인**으로 연결합니다. "앱 삭제 후 3일 내 경쟁앱 설치자"처럼 **행동 기반으로 타겟을 정의**하고, 모든 오디언스 모수는 **AWS Athena로 DMP 교차 시그널을 실측**합니다.

## ③ AI(Kiro) 활용 방식

이 프로젝트는 **설계부터 운영까지 전 과정에서 Kiro를 활용**했습니다. 단순 코드 생성이 아니라, 아키텍처 설계, 데이터 파이프라인 구축, 품질 관리, 디버깅까지 Kiro와 함께 진행했습니다.

### 1. 시스템 설계 및 코드 생성

**아키텍처 설계**: 트렌드 수집 → Athena 실측 → LLM 생성 → 정적 사이트 빌드의 전체 파이프라인을 Kiro와 설계. AWS Bedrock/Anthropic/OpenAI 멀티 LLM 폴백 구조도 Kiro가 제안.

**코드 생성 (총 1,568줄)**: `daily_publish.py`(625줄), `build_static.py`(355줄), `app.py`(588줄)를 Kiro와 함께 작성. Google News RSS 파싱, Athena 교차 쿼리, Claude 프롬프트 구성, GitHub Pages 빌드까지 전체 파이프라인 코드.

**프롬프트 엔지니어링**: 에세이 3막 구조(상황→타겟 정의→전환), DMP 추적 가능 시그널만 사용하는 제약 조건, 3타입 템플릿 로테이션을 Kiro와 설계.

### 2. 디버깅 및 문제 해결 (구체적 사례)

**DMP 쿼리 권한 디버깅**: Athena 쿼리가 AccessDenied로 실패했을 때, Kiro가 `daily_publish.py` 소스를 분석해 `WorkGroup="mi.ai"` 파라미터가 빠져있음을 발견. 이 한 줄이 권한 문제의 원인이었습니다.

**DMP 카테고리 매칭 수정**: 여행/반려동물 오디언스 모수가 0으로 나오는 문제를 Kiro가 진단. DMP 테이블의 실제 카테고리명(`%숙박%` → `%호텔%`, `%반려%` → `%애완동물%`)을 Athena로 조회해 쿼리 조건을 수정.

**중복 에세이 자동 식별**: 25개 에세이의 제목·본문·audience-box를 Kiro가 크로스 분석해 13개 중복을 식별. 같은 카테고리에서 LLM이 동일 패턴의 제목을 반복 생성하는 근본 원인까지 진단.

**에세이 구조 일괄 통일**: 12개 에세이의 HTML 섹션 순서가 3가지 패턴으로 섞여있는 것을 Kiro가 탐지. 정규식으로 quote/punchline 블록을 자동 스왑하는 스크립트를 작성해 전체를 한 번에 통일.

**제목 불일치 탐지**: `app.py`, `build_static.py`, `essays_html/` 3곳에 저장된 제목이 서로 다른 문제를 Kiro가 자동 비교 스크립트로 탐지하고 동기화.

### 3. 매일 자동 발행 (Claude API)

매일 10시 크론잡 실행 시:
1. Google News RSS에서 13개 카테고리별 뉴스 수집 → 볼륨 기반 카테고리 자동 선정
2. AWS Athena로 교차 시그널 ADID 수 실측 (예: "운동앱 2개 + 쇼핑앱 동시 사용자 = ~89만 명")
3. Claude API로 행동 기반 오디언스 에세이 자동 생성
4. 정적 사이트 재빌드 → git push → GitHub Pages 자동 배포

## ④ 기대 효과

### 며칠 걸리던 일이 90초로

기존에 라이브 오디언스를 만들고 마케팅 리포트로 기획하려면, 사람이 직접 트렌드를 조사하고 DMP 데이터를 분석하고 제안서를 작성해야 했습니다.

오늘의 오디언스는 이 전체 과정을 **90초**로 압축합니다.

| 단계 | 기존 (수동) | 자동화 후 |
|---|---|---|
| 트렌드 리서치 | 뉴스·커뮤니티 모니터링 (2~3시간) | RSS 자동 수집 (30초) |
| 주제 선정 | 팀 회의, 광고주 니즈 파악 (반나절) | 뉴스 볼륨 기반 자동 선정 (즉시) |
| DMP 데이터 분석 | 분석가가 쿼리 작성·실행 (3~4시간) | Athena 자동 실측 (1분) |
| 오디언스 정의 | 기획자가 스토리 구성 (3~4시간) | Claude 에세이 자동 생성 (2분) |
| 리포트 디자인·배포 | 디자이너 작업 + 공유 (반나절~1일) | 자동 빌드·배포 (즉시) |
| **합계** | **2~3일 (여러 직군 투입)** | **~90초 (무인 자동)** |

### 이것이 만드는 변화

1. **속도**: 트렌드가 터지면 다음 날 아침에 바로 오디언스 제안이 나옵니다.
2. **인력 효율**: 기획자·분석가·디자이너가 각각 투입되던 작업을 파이프라인 하나로 대체합니다.
3. **타겟 품질**: "2030 여성" 대신 "앱 삭제 후 3일 내 경쟁앱 설치자"처럼 행동 기반으로 정의합니다.
4. **데이터 신뢰**: 모든 오디언스 모수는 DMP Athena 실측값입니다. 추정치가 아닙니다.
5. **영업 활용**: AE가 광고주 미팅 전 최신 에세이를 공유하면, DMP 가치를 즉시 보여줄 수 있습니다.

## 부록: 비즈니스 구조 설계

### 전체 구조

```
[콘텐츠]                  [전환]                    [매출]
오늘의 오디언스      →   구독 / 문의 / 미팅    →   DMP 상품 계약
(매일 자동 발행)          (리드 확보)                (오디언스 팩 판매)
```

에세이 자체가 DMP 상품의 쇼케이스. 광고주가 "이 오디언스를 내 캠페인에 쓰고 싶다"고 먼저 찾아오는 인바운드 세일즈 구조.

### 상품 구조

| 상품 | 모델 | 설명 |
|---|---|---|
| 오디언스 팩 | 건당 과금 | 에세이의 오디언스를 실제 ADID 세그먼트로 제공, DSP 연동 |
| 커스텀 오디언스 | 프로젝트 과금 | 광고주 맞춤 조건으로 DMP 교차 분석 → 세그먼트 생성 |
| 오디언스 구독 | 월정액 | 매일 새 오디언스 자동 업데이트 + DSP 실시간 연동 |

### 세일즈 퍼널 비교

```
기존:  콜드콜 → 제안서 → 미팅 → 데모 → 계약 (3~6개월)
신규:  에세이 발견 → "이거 쓰고 싶다" → 미팅 → 계약 (2~4주)
```
