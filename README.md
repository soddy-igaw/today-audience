# 오늘의 오디언스

> 매일 아침, AI가 트렌드를 읽고 DMP 행동 데이터를 교차 분석해 — 광고주가 바로 쓸 수 있는 오디언스를 자동으로 제안합니다.

🔗 **라이브**: https://soddy-igaw.github.io/today-audience/

## ① 프로젝트 소개

**오늘의 오디언스**는 IGAWorks의 DMP(Data Management Platform) 행동 데이터와 실시간 뉴스 트렌드를 결합해, 광고 마케터·AE·광고주가 즉시 활용할 수 있는 타겟 오디언스를 매일 자동 생성·발행하는 시스템입니다.

### 핵심 파이프라인

```
뉴스 트렌드 수집 → 카테고리 자동 선정 → DMP 교차 시그널 실측(Athena)
→ AI 에세이 생성(Claude) → 코드 반영 → 정적 사이트 빌드 → GitHub Pages 배포
```

### 구성 요소

| 파일 | 역할 |
|---|---|
| `daily_publish.py` | 자동 발행 파이프라인 (트렌드 수집 → LLM 에세이 생성 → 코드 반영) |
| `build_static.py` | 정적 사이트 빌드 (GitHub Pages용 `docs/` 생성) |
| `app.py` | Streamlit 대시보드 (내부 프리뷰용) |
| `run_daily.sh` | 크론잡 실행 스크립트 (매일 10시) |
| `templates/` | 에세이 3타입 HTML 템플릿 (감정형·인사이트형·데이터형) |

### 커버 카테고리 (13개)

금융, 부동산, 증권, 쇼핑, 여행, 헬스, 자동차, 게임, 골프, 반려동물, 교육, 패션, 배달

## ② 문제 정의

### 광고 업계의 오디언스 기획 병목

광고 대행사와 광고주는 캠페인마다 "누구에게 보여줄 것인가"를 정해야 합니다. 이 과정에서 반복되는 문제:

1. **시간 소모**: 오디언스 하나를 기획하는 데 트렌드 리서치 2시간 + 데이터 분석 3시간 + 제안서 작성 3시간 = **평균 1~2일 소요**
2. **트렌드 반영 지연**: 뉴스가 터져도 오디언스 제안까지 최소 3~5일 걸려 타이밍을 놓침
3. **추상적 타겟 정의**: "2030 여성", "관심사: 여행" 같은 인구통계 기반 타겟은 실제 전환율이 낮음
4. **데이터 단절**: 트렌드 분석하는 사람과 DMP 데이터를 다루는 사람이 분리되어 있어 교차 인사이트가 나오기 어려움

### 이 프로젝트가 해결하는 것

- 뉴스 트렌드 → DMP 행동 시그널 → 오디언스 제안을 **하나의 자동화 파이프라인**으로 연결
- "앱 삭제 후 3일 내 경쟁앱 설치" 같은 **행동 기반 타겟 정의**로 전환율 향상
- 매일 아침 자동 발행으로 **트렌드 반영 지연 제로**

## ③ AI(Kiro) 활용 방식

### 1단계: 프로젝트 설계 및 초기 구축 (Kiro)

Kiro를 활용해 전체 시스템을 설계하고 구축했습니다:

- **아키텍처 설계**: 트렌드 수집 → LLM 생성 → 정적 사이트 빌드의 파이프라인 구조 설계
- **코드 생성**: `daily_publish.py`(625줄), `build_static.py`(355줄), `app.py`(588줄) — 총 1,568줄의 코드를 Kiro와 함께 작성
- **프롬프트 엔지니어링**: 에세이 3막 구조(상황→타겟 정의→전환), 행동 기반 제목 규칙, DMP 추적 가능 시그널만 사용하는 제약 조건 등 상세 프롬프트 설계
- **디자인 시스템**: 토스 피드 스타일의 에세이 HTML/CSS 템플릿 3종(감정형·인사이트형·데이터형) 제작
- **자동화 구축**: GitHub Actions 워크플로우, 크론잡 설정, AWS Bedrock/Anthropic/OpenAI 멀티 LLM 폴백 구조
- **품질 관리**: 중복 에세이 탐지 및 정리, 에세이 흐름(인터뷰→펀치라인) 통일, 제목 일관성 검증 등 유지보수 작업

### 2단계: 매일 자동 발행 (Claude API)

매일 10시 크론잡이 실행되면:

1. **트렌드 수집**: Google News RSS에서 13개 카테고리별 최신 뉴스 수집, 뉴스 수가 가장 많은 카테고리 자동 선정
2. **DMP 실측**: AWS Athena로 교차 시그널 ADID 수 실측 (예: "운동앱 2개 + 쇼핑앱 동시 사용자 = ~89만 명")
3. **에세이 생성**: Claude API로 행동 기반 오디언스 에세이 자동 생성
4. **빌드 & 배포**: 정적 사이트 재빌드 → git push → GitHub Pages 자동 배포

### Kiro가 특히 효과적이었던 부분

- **반복 작업 자동화**: 에세이 HTML 템플릿 3종을 만들 때, 하나의 디자인을 잡고 변형을 빠르게 생성
- **디버깅**: `daily_publish.py`의 제목 추출 정규식이 toss 스타일 HTML에서 실패하는 문제를 진단하고 수정
- **대규모 리팩토링**: 25개 에세이 중 중복 13개를 식별하고, 섹션 순서를 일괄 통일하는 작업을 한 세션에서 완료
- **크로스 파일 일관성**: `app.py`, `build_static.py`, `essays_html/` 간 제목·데이터 불일치를 탐지하고 동기화

### 광고 업계

| 기존 | 오늘의 오디언스 |
|---|---|
| 오디언스 기획 1~2일 | **10분 내 자동 생성** |
