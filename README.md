## ① 프로젝트 소개

**오늘의 오디언스**는 IGAWorks의 DMP(Data Management Platform) 행동 데이터와 실시간 뉴스 트렌드를 결합해, 광고 마케터·AE·광고주가 즉시 활용할 수 있는 타겟 오디언스를 매일 자동 생성·발행하는 시스템입니다.

### 핵심 파이프라인

```
뉴스 트렌드 수집 → 카테고리 자동 선정 → DMP 교차 시그널 실측(Athena)
→ AI 에세이 생성(Claude) → 코드 반영 → 정적 사이트 빌드 → GitHub Pages 배포
```

### 구성 요소

| 파일 | 역할 |
|---|---|
| `daily_publish.py` | 자동 발행 파이프라인 (트렌드 수집 → LLM 에세이 생성 → 코드 반영) |
| `build_static.py` | 정적 사이트 빌드 (GitHub Pages용 `docs/` 생성) |
| `app.py` | Streamlit 대시보드 (내부 프리뷰용) |
| `run_daily.sh` | 크론잡 실행 스크립트 (매일 10시) |
| `templates/` | 에세이 3타입 HTML 템플릿 (감정형·인사이트형·데이터형) |

### 커버 카테고리 (13개)

금융, 부동산, 증권, 쇼핑, 여행, 헬스, 자동차, 게임, 골프, 반려동물, 교육, 패션, 배달

## ② 문제 정의

### 광고 업계의 오디언스 기획 병목

광고 대행사와 광고주는 캠페인마다 "누구에게 보여줄 것인가"를 정해야 합니다. 이 과정에서 반복되는 문제:

1. **시간 소모**: 오디언스 하나를 기획하는 데 트렌드 리서치 2시간 + 데이터 분석 3시간 + 제안서 작성 3시간 = **평균 1~2일 소요**
2. **트렌드 반영 지연**: 뉴스가 터져도 오디언스 제안까지 최소 3~5일 걸려 타이밍을 놓침
3. **추상적 타겟 정의**: "2030 여성", "관심사: 여행" 같은 인구통계 기반 타겟은 실제 전환율이 낮음
4. **데이터 단절**: 트렌드 분석하는 사람과 DMP 데이터를 다루는 사람이 분리되어 있어 교차 인사이트가 나오기 어려움

### 이 프로젝트가 해결하는 것

- 뉴스 트렌드 → DMP 행동 시그널 → 오디언스 제안을 **하나의 자동화 파이프라인**으로 연결
- "앱 삭제 후 3일 내 경쟁앱 설치" 같은 **행동 기반 타겟 정의**로 전환율 향상
- 매일 아침 자동 발행으로 **트렌드 반영 지연 제로**

## ③ AI(Kiro) 활용 방식

### 1단계: 프로젝트 설계 및 초기 구축 (Kiro)

Kiro를 활용해 전체 시스템을 설계하고 구축했습니다:

- **아키텍처 설계**: 트렌드 수집 → LLM 생성 → 정적 사이트 빌드의 파이프라인 구조 설계
- **코드 생성**: `daily_publish.py`(625줄), `build_static.py`(355줄), `app.py`(588줄) — 총 1,568줄의 코드를 Kiro와 함께 작성
- **프롬프트 엔지니어링**: 에세이 3막 구조(상황→타겟 정의→전환), 행동 기반 제목 규칙, DMP 추적 가능 시그널만 사용하는 제약 조건 등 상세 프롬프트 설계
- **디자인 시스템**: 토스 피드 스타일의 에세이 HTML/CSS 템플릿 3종(감정형·인사이트형·데이터형) 제작
- **자동화 구축**: GitHub Actions 워크플로우, 크론잡 설정, AWS Bedrock/Anthropic/OpenAI 멀티 LLM 폴백 구조
- **품질 관리**: 중복 에세이 탐지 및 정리, 에세이 흐름(인터뷰→펀치라인) 통일, 제목 일관성 검증 등 유지보수 작업

### 2단계: 매일 자동 발행 (Claude API)

매일 10시 크론잡이 실행되면:

1. **트렌드 수집**: Google News RSS에서 13개 카테고리별 최신 뉴스 수집, 뉴스 수가 가장 많은 카테고리 자동 선정
2. **DMP 실측**: AWS Athena로 교차 시그널 ADID 수 실측 (예: "운동앱 2개 + 쇼핑앱 동시 사용자 = ~89만 명")
3. **에세이 생성**: Claude API로 행동 기반 오디언스 에세이 자동 생성
4. **빌드 & 배포**: 정적 사이트 재빌드 → git push → GitHub Pages 자동 배포

### Kiro가 특히 효과적이었던 부분

- **반복 작업 자동화**: 에세이 HTML 템플릿 3종을 만들 때, 하나의 디자인을 잡고 변형을 빠르게 생성
- **디버깅**: `daily_publish.py`의 제목 추출 정규식이 toss 스타일 HTML에서 실패하는 문제를 진단하고 수정
- **대규모 리팩토링**: 25개 에세이 중 중복 13개를 식별하고, 섹션 순서를 일괄 통일하는 작업을 한 세션에서 완료
- **크로스 파일 일관성**: `app.py`, `build_static.py`, `essays_html/` 간 제목·데이터 불일치를 탐지하고 동기화

## ④ 기대 효과

### 핵심: 라이브 오디언스 생성의 자동화

기존에 라이브 오디언스를 만들고 마케팅 리포트로 기획하려면, 사람이 직접 트렌드를 조사하고 DMP 데이터를 분석하고 제안서를 작성해야 했습니다. 이 과정이 **최소 2~3일**, 복잡한 경우 일주일 이상 소요됩니다.

오늘의 오디언스는 이 전체 과정을 **트렌드 수집 → AI 자동 생성 → 즉시 배포**로 압축합니다.

| 단계 | 기존 (수동) | 자동화 후 |
|---|---|---|
| 트렌드 리서치 | 담당자가 뉴스·커뮤니티 모니터링 (2~3시간) | Google News RSS 자동 수집 (30초) |
| 카테고리·주제 선정 | 팀 회의, 광고주 니즈 파악 (반나절) | 뉴스 볼륨 기반 자동 선정 (즉시) |
| DMP 데이터 분석 | 데이터 분석가가 쿼리 작성·실행 (3~4시간) | Athena 교차 시그널 자동 실측 (1분) |
| 오디언스 정의 | 기획자가 타겟 정의·스토리 구성 (3~4시간) | Claude가 행동 기반 에세이 자동 생성 (2분) |
| 리포트 디자인·배포 | 디자이너 작업 + 공유 (반나절~1일) | 정적 사이트 자동 빌드·배포 (즉시) |
| **합계** | **2~3일 (여러 직군 투입)** | **~4분 (무인 자동)** |

### 이것이 의미하는 것

1. **속도**: 트렌드가 터지면 다음 날 아침에 바로 오디언스 제안이 나옵니다. 기존에는 트렌드가 식은 뒤에야 리포트가 완성되던 문제를 해결합니다.

2. **인력 효율**: 기획자·데이터 분석가·디자이너가 각각 투입되던 작업을 자동화 파이프라인 하나로 대체합니다. 사람은 최종 검수와 광고주 커뮤니케이션에 집중할 수 있습니다.

3. **타겟 품질**: "2030 여성" 같은 인구통계 타겟 대신, "앱 삭제 후 3일 내 경쟁앱 설치자"처럼 DMP 행동 시그널 기반으로 정의하므로 광고 전환율 향상이 기대됩니다.

4. **영업 활용**: AE가 광고주 미팅 전 최신 오디언스 에세이를 공유하면, "우리 DMP로 이런 타겟팅이 가능합니다"를 구체적으로 보여줄 수 있습니다. 추상적인 데이터 플랫폼 소개 대신, 읽을 수 있는 인사이트로 DMP 가치를 전달합니다.

5. **리드 확보**: 이메일 구독 기능을 통해 잠재 광고주를 자연스럽게 모을 수 있는 콘텐츠 마케팅 채널로 기능합니다.

## ⑥ 비즈니스 구조 설계

### 전체 구조

```
[콘텐츠 레이어]              [전환 레이어]              [매출 레이어]

오늘의 오디언스         →   리드 확보 / 관계 형성   →   DMP 상품 계약
(매일 자동 발행)             (구독, 문의, 미팅)          (오디언스 팩 판매)
```

에세이 자체가 DMP 상품의 쇼케이스입니다. 광고주가 에세이를 읽고 "이 오디언스를 내 캠페인에 쓰고 싶다"고 먼저 찾아오는 인바운드 세일즈 구조.

### Phase 1: 트래픽 & 브랜딩 (현재 ~ 1개월)

**목표**: 광고 업계에서 매일 보는 콘텐츠로 자리잡기

| 채널 | 방식 |
|---|---|
| 웹사이트 | 매일 자동 발행 (현재 운영 중) |
| 이메일 뉴스레터 | 구독자에게 매일 아침 발송 (구독 폼 구축 완료) |
| 링크드인/브런치 | 에세이 요약본 주 2~3회 포스팅 → 웹 유입 |
| AE 영업 도구 | 광고주 미팅 전 최신 에세이 공유 |

### Phase 2: 리드 전환 (1~3개월)

**목표**: 콘텐츠 소비자를 DMP 잠재 고객으로 전환

**전환 퍼널**:

```
에세이 조회 (마케터, 광고주, 일반인)
    ↓
구독 (이메일 — 매일 받아보는 사람)
    ↓
샘플 요청 ("이 오디언스 실제 모수/구성 알려주세요")
    ↓
미팅 (AE가 DMP 상품 소개)
    ↓
계약 (오디언스 팩 or DMP 월정액)
```

### Phase 3: 매출 구조 (3개월~)

#### 상품 1. 오디언스 팩 (건당 과금)

에세이에 소개된 오디언스를 실제 광고에 쓸 수 있는 세그먼트로 제공합니다. 에세이가 "무료 샘플"이고, ADID 리스트가 "유료 상품"입니다.

```
[에세이]                                    [상품]
"전기차 포기하고 하이브리드 찾는 사람"   →   ADID 리스트 75만 건, DSP 연동 가능
```

#### 상품 2. 커스텀 오디언스 (프로젝트 과금)

광고주가 원하는 조건으로 맞춤 오디언스를 설계합니다.

```
광고주: "우리 경쟁사 앱 삭제 후 우리 앱 미설치자 찾아주세요"
    → DMP 교차 분석 → 커스텀 세그먼트 생성
```

#### 상품 3. 오디언스 구독 (월정액)

매일 생성되는 오디언스를 DSP에 자동 연동하는 구독 모델입니다.

```
- 매일 새 오디언스 자동 업데이트
- DSP 실시간 연동
- 전용 대시보드
```

### 핵심: 콘텐츠가 세일즈 퍼널을 대체

```
기존 DMP 영업:  콜드콜 → 제안서 → 미팅 → 데모 → 계약 (3~6개월)
오늘의 오디언스: 에세이 발견 → "이거 쓰고 싶다" → 미팅 → 계약 (2~4주)
```
